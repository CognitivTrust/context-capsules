# Image Manifest — Context Capsules README Assets

Two kinds of assets live here, made by two different processes, on purpose:

- **Information diagrams (`.svg`, 9 files)** are hand-authored directly as SVG markup by the lead designer (not generated by a diffusion model). Diagrams that need precise structure and legible text labels — architecture, lifecycle, comparisons, terminal mockups, icons — are far more reliable as code than as AI-generated raster art, and SVG is infinitely scalable and trivially themeable. This is also why they are not literally "AI-generated images": they are deterministic, reviewable vector code, which is the most reliable way to guarantee an information-design (Stripe/Linear/Excalidraw-style) result rather than a decorative one.
- **Brand assets (`.png`, 2 files)** — the top-of-README hero and the closing/social-preview banner — went through a two-agent pipeline: a Gemini 3.1 Pro designer sub-agent generated candidates against a strict flat/no-glow/no-gradient brief, and a Gemini 3.5 Flash reviewer sub-agent independently inspected the rendered pixels against a rejection checklist (no glow, no gradient-mesh, no "AI art" look, must read as a flat Stripe/Linear-style icon).

All colors, stroke widths, corner radii, and type choices share one palette across every asset:
`indigo #6366F1` · `teal #0D9488` · `violet #7C3AED` · `amber #D97706` · `sky #0284C7` · `ink #0F172A` · `muted #64748B` · `border #E2E8F0`.

**Independent QA pass:** a Gemini 3.5 Flash reviewer sub-agent audited all 11 assets against a strict "explains something / no AI-art look / matches the design system" checklist and flagged two issues, both since fixed:

1. `icon-multi-agent.svg` was vertically off-center (top-heavy) and used a lone window dot. Fixed by re-centering the three window shapes around the icon's vertical midpoint and adding a three-dot terminal header to match `cli-terminal.svg`'s window chrome.
2. `social-preview.png` had a faint residual left-to-right highlight on each bar instead of a perfectly flat fill — a known limitation of the diffusion model even after 3 prompt-revision rounds. Rather than spend further generation attempts chasing a flat fill probabilistically, the bar regions were detected and redrawn deterministically with exact brand hex values (`#6366F1` / `#0D9488` / `#7C3AED` / `#D97706` on `#0F172A`), guaranteeing pixel-perfect flat color.

Every asset is now ACCEPT against the reviewer's checklist.

## Generated brand assets (Gemini 3.1 Pro designer + Gemini 3.5 Flash reviewer)

### `hero-banner.png`
- **Final prompt used** (3rd attempt, after fixing an off-brief first pass and a "play button" misread on the terminal glyph):
  > Wide 1600x500 flat vector banner illustration, Stripe/Linear marketing-site style. Solid near-white background (#F8FAFC), absolutely no gradients anywhere, no glow, no blur, no drop shadows. Three simple flat geometric icons arranged left-to-right at generous spacing, each a clean rounded-square outline badge with a thin 3px stroke border and transparent/white interior: (1) a minimal git-branch glyph made of two small filled circles connected by a single forked line, solid slate gray (#475569) stroke and fill, (2) a minimal command-line terminal glyph: a rounded-rectangle outline with a small solid right-pointing chevron/angle-bracket ">" shape and a short solid horizontal underscore bar next to it (a command prompt cursor, NOT a play/video triangle), solid indigo (#6366F1), (3) a stack of exactly 3 flat solid horizontal rounded bars of decreasing width stacked with small gaps, representing a layered ledger, solid teal (#0D9488). Thin flat solid arrows (slate gray #475569, uniform 3px stroke, no gradient, no glow) connect badge 1 to badge 2 to badge 3 in a simple left-to-right flow. Strictly flat 2D vector shapes, crisp solid edges, zero shadow, zero glow, zero gradient, zero blur, zero 3D, no text, no logos, no faces, generous whitespace, confident minimal icon row exactly like a Stripe.com or Linear.app marketing hero section.
- **Alt text:** "Three flat outlined icon badges in a left-to-right flow: a git-branch glyph, a terminal command-prompt glyph, and a stack of layered ledger bars, connected by arrows."
- **Used:** top of README, directly under the title and tagline.
- **QA result:** ACCEPT. Flat solid colors, crisp edges, no glow/gradient/shadow, reads instantly as "history → CLI → ledger."

### `social-preview.png`
- **Final prompt used** (3rd attempt):
  > Minimalist flat 2D vector graphic, NOT a photo, NOT 3D render, NOT glossy. Image size 1280x640. Background: one single flat matte color #0F172A filling the entire canvas uniformly, like a CSS background-color, zero variation. In the center, 4 separate rectangle shapes stacked vertically with small gaps, each rectangle has slightly rounded corners and is filled with exactly one flat matte CSS-style background-color and that single color only, like swatches in a flat design system: rectangle 1 = #6366F1, rectangle 2 = #0D9488, rectangle 3 = #7C3AED, rectangle 4 = #D97706. Render this like vector clip-art / a Figma flat-fill rectangle, not like a rendered 3D object — zero lighting, zero highlight, zero gloss, zero shine, zero gradient, zero shadow, zero glow. Pure flat color blocks only. Generous empty space around the stack. No text, no logo, no icons besides the rectangles, no faces.
- **Alt text:** "Four flat solid-colored bars — indigo, teal, violet, and amber — stacked vertically on a dark background, representing a layered append-only ledger."
- **Used:** closing banner before the License section; suggested as the GitHub repo social-preview image (Settings → General → Social preview).
- **QA result:** the reviewer flagged the diffusion-generated version for a faint residual highlight on each bar. Fixed deterministically: the four bar regions were detected by pixel sampling and redrawn as exact flat fills (`#6366F1` / `#0D9488` / `#7C3AED` / `#D97706` on `#0F172A`) with Pillow, so the shipped file has zero gradient, by construction rather than by chance.
